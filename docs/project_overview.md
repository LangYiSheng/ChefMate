# ChefMate 项目简介

**ChefMate** 是一个面向日常家庭烹饪场景的智能烹饪助手，目标是为用户提供从“**今天做什么菜**”到“**食材是否足够**”、再到“**缺什么、怎么买、怎么做**”的一体化智能服务。系统结合 **AI 大模型** 与 **多工具协同调用能力**，以类似 ChatGPT 的对话式交互体验，为用户提供一个能够持续陪伴、理解需求、组织流程并主动辅助完成烹饪任务的个人助手。

与传统菜谱应用不同，ChefMate 不只是被动展示菜谱，而是通过智能体驱动完整流程，结合用户偏好、健康需求、现有厨具、已有食材等信息，动态生成推荐、判断材料可行性、辅助采购，并在烹饪过程中提供结构化指导与实时问答支持，形成完整闭环。

# 系统核心定位

ChefMate 的核心定位是一个“**会思考、会调用工具、会分步骤帮助用户完成做饭任务的智能体助手**”。

用户不需要先自己决定做什么菜，也不需要自己手动在多个页面之间切换。用户只需要像和 ChatGPT 聊天一样发起一个新对话，例如：

> “今晚吃什么比较好？”

此时，智能体会理解用户意图，并自动调用后端相关工具与服务，完成推荐、筛选、确认、采购辅助、烹饪指导等多个环节的协同处理，再将结果以 **大模型自然语言总结 + 结构化功能卡片** 的方式展示给用户，提升整个使用过程的清晰度、交互感和可操作性。

# 整体业务流程说明

ChefMate 围绕用户的做饭过程，设计了一个完整的智能决策与辅助流程，整体上可分为以下几个阶段：

## 1. 做什么菜：智能推荐阶段

当用户进入系统后，可以从“想做什么菜”这一问题出发，由系统帮助完成决策。

智能体会调用推荐系统相关服务，综合分析以下信息：

* 用户个人喜好
* 用户健康需求
* 用户已有厨具
* 用户当前已有食材

在此基础上，系统生成一组适合当前用户情境的推荐菜品列表。
智能体不会只机械返回结果，而是会结合大语言模型能力，对推荐结果进行总结说明，例如推荐理由、适合场景、口味特点、制作难度等，帮助用户更自然地做出选择。

如果用户已经明确表示想吃某种菜，系统也可以直接围绕该菜品展开后续流程，而不必重复推荐。

## 2. 有什么食材：食材管理阶段

在用户选定目标菜品后，系统进入食材判断阶段。此时智能体会调用食材管理相关工具，对当前做这道菜所需材料与用户已有食材进行比对，判断：

* 是否已有足够食材
* 缺少哪些原材料
* 是否存在可替代食材
* 当前已有食材是否适合优先使用

如果用户本身已经知道自己手头有哪些食材，也可以直接从“我已经有这些食材”这一入口出发，让系统反向推荐适合制作的菜品。

此外，食材管理模块还支持通过**拍照识别**方式辅助录入食材信息，降低用户手动输入成本，让系统更容易掌握“冰箱里到底还有什么”。

## 3. 缺什么材料：采购辅助阶段

当系统发现食材不足时，智能体会继续推进流程，自动分析出“还需要购买哪些原材料”。

这一阶段不是简单列清单，而是通过更适合移动端或对话场景的结构化卡片进行展示。例如可以采用类似“单词卡”的卡片形式，逐项展示用户需要采购的食材，使信息更清晰、直观、便于快速查看。

在采购辅助过程中，系统还可以进一步为用户提供挑选支持，例如：

* 推荐采购数量
* 说明食材用途
* 提供替代选项
* 调用拍照/识别相关工具辅助判断食材新鲜度

例如用户在买菜时可以拍摄蔬菜、肉类或水果图片，系统调用新鲜度检测工具，对食材品质进行辅助判断，从而帮助用户更合理地完成选购。

图中的“果蔬新鲜度分类”就属于采购辅助能力的一部分，它不是独立终点，而是服务于“买什么、怎么买得更好”这一场景。

## 4. 怎么做：烹饪指导阶段

当所需材料已经齐备后，系统进入“做饭流程”阶段。此时智能体调用烹饪指导生成工具，为目标菜品生成一套结构化的烹饪指导内容。

这套内容不是普通长篇菜谱，而应以“**指导书卡片**”形式进行展示，具备以下特点：

* 步骤清晰，按顺序逐步展开
* 每一步都描述具体操作内容
* 用户可以逐步查看，不会信息过载
* 卡片内可附带时间、火候、顺序等重点提示
* 支持自动定时提醒等辅助功能

例如煎、煮、蒸、炖等步骤中，系统可以自动提供计时提醒，减少用户遗漏关键节点的风险。这样用户在真正做饭时，不是“看完一篇菜谱然后自己记”，而是跟着系统一步一步完成。

## 5. 边做边问：烹饪过程中的持续陪伴

ChefMate 的一个重要特点是：烹饪指导并不是静态结束，而是一个**持续交互的过程**。

在指导书卡片展示过程中，智能体仍然持续在线，用户可以随时提问，例如：

* “这个火是不是开大了？”
* “现在颜色这样正常吗？”
* “没有生抽能不能换别的？”
* “这一步炒多久合适？”

系统会基于当前菜品上下文、当前步骤信息以及大模型理解能力，继续为用户提供实时帮助。这意味着 ChefMate 不只是“推荐系统 + 菜谱系统”的拼接，而是一个真正具备过程陪伴能力的智能助手。

## 6. 可选扩展：菜谱上传与分享

除核心闭环外，系统还支持一个可选扩展流程：
用户可以上传自己的菜品内容，系统对步骤进行录制或整理，最终生成可分享的菜谱链接，实现用户自制内容的沉淀与传播。

这一部分对应图中的可选区域，包括：

* 上传菜品
* 步骤录制
* 上传共享 / 分享链接

它可以作为系统的扩展功能，为后续社区化、内容生态化提供基础，但不影响主流程的成立。

# 图中流程的文字化表达

根据流程图，ChefMate 的业务逻辑可以概括为：

用户进入系统后，可以从“想做某种菜”或“已经有某种食材”两个方向发起需求。
如果用户尚未确定菜品，则由推荐系统结合个人喜好、健康需求、已有厨具和已有食材生成推荐菜品；如果用户已经明确目标菜品，则直接进入食材管理阶段。

食材管理模块负责接收用户已有食材信息，并支持拍照识别等方式辅助录入。系统判断做目标菜所需材料是否充足：
若材料不足，则进一步分析缺失项，进入采购原材料辅助流程，并可结合果蔬新鲜度分类等能力帮助用户选购；若材料足够，则直接进入做饭流程。

在做饭流程中，系统生成结构化步骤指导，并通过流程指导卡片向用户逐步展示，同时提供自动提醒等能力。在整个烹饪过程中，用户仍可继续向智能体发问，智能体持续提供帮助，直到完成本次烹饪任务。

此外，系统还支持上传菜品、步骤录制及分享链接生成等可选扩展功能，用于形成用户内容沉淀与分享能力。

# 典型交互示例

下面是一个典型使用场景：

用户新建一个对话，输入：

> “今晚吃什么比较好？”

智能体首先识别出用户意图是“菜品推荐”，于是调用推荐系统服务，根据用户偏好、健康需求、已有厨具和现有食材生成一组候选菜品。
随后，智能体用自然语言对推荐结果做总结，并配合预制菜品卡片展示给用户。

用户从中选择一道菜后，智能体继续调用食材管理工具，检查用户当前是否具备制作该菜所需的原材料。
如果发现部分食材缺失，系统会生成采购卡片，逐项列出需要购买的食材；用户在采购时还可以通过拍照调用新鲜度检测工具，获取挑选辅助建议。

当食材准备完成后，智能体再调用烹饪指导生成工具，为用户输出一套结构化“指导书”卡片。
用户可以按步骤逐步查看做法，系统还会根据步骤内容自动提供定时提醒等服务。
在用户做菜过程中，如果遇到任何问题，仍然可以继续在当前对话中向智能体提问，系统会结合当前上下文持续提供帮助，直到用户完成整道菜的制作。

# 交互呈现方式

ChefMate 的结果展示应采用“**大模型文本 + 工具卡片**”结合的方式：

* **大模型文本**：负责理解用户需求、总结推荐结果、解释原因、回答过程问题，提供类似 ChatGPT 的自然交流体验
* **工具卡片**：负责承载结构化结果，例如推荐菜品卡片、采购清单卡片、指导书卡片、提醒卡片等

这种组合方式既保留了自然语言交互的灵活性，也保证了信息展示的结构化和操作效率，更适合复杂多阶段任务的引导。
