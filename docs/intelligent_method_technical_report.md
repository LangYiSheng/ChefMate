# ChefMate 智能方法技术报告

## 1. 项目当前智能方法概况

ChefMate 当前涉及三类智能方法：

1. 对话理解与任务推进
2. 菜谱推荐
3. 食材图像识别

当前进展概况如下：

| 智能方法 | 当前状态 | 说明 |
| --- | --- | --- |
| 对话理解与任务推进 | 已进入主线开发 | 已基于 `FastAPI + LangChain + LangGraph` 实现多阶段任务推进、工具调用、流式消息与结构化卡片联动 |
| 菜谱推荐 | 已实现基础能力 | 当前主要是基于 MySQL 菜谱库的 SQL 查询、标签过滤、食材匹配与简单规则推荐 |
| 食材图像识别 | 双路线均有成果 | 已接入多模态大模型识别路线，同时保留并完成了 YOLO 路线的阶段性训练与评测 |

## 2. 智能方法一：对话理解与任务推进

### 2.1 当前实现功能

该方法是 ChefMate 的核心智能方法，目标是让系统不只是“聊天”，而是围绕一次真实做饭任务完成状态推进。

当前已经实现的能力包括：

- 识别用户当前意图处于“闲聊、推荐菜品、备料、烹饪”中的哪一类
- 创建活动任务并维护任务阶段
- 通过工具更新用户记忆、任务状态、菜谱步骤与食材状态
- 通过 LangGraph 执行 ReAct 流程
- 返回消息并附带结构化卡片

系统的任务主要工作流为：
idea -> planning -> shopping -> cooking -> idea

### 2.2 当前实现方式

该方法当前采用“大模型 API + Prompt + ReAct + LangGraph + 任务状态管理”的组合实现。

其中：

- `LangChain` 用于模型调用、Prompt 模板与工具抽象
- `LangGraph` 用于工作流编排
- `FastAPI` 用于消息接口
- `MySQL` 用于持久化用户、对话、任务、消息

## 3. 智能方法二：菜谱推荐

### 3.1 当前实现功能

智能方法二由于时间所限，当前实现得相对朴素，主要还是围绕菜谱库做 SQL 查询与规则过滤。

当前能力包括：

- 菜名检索
- 标签过滤
- 基于已有食材匹配菜谱
- 根据长期记忆标签做默认推荐

### 3.2 当前方法特点

这一部分当前还谈不上独立的训练式智能方法，更多是：

- 结构化数据库查询
- 标签字典匹配
- 简单规则打分

因此它目前还不具备独立模型评测体系，也还没有形成专门的数据集测试结果。

### 3.3 当前结论

智能方法二目前已经能支撑产品主流程，但当前阶段仍不是一个训练式智能推荐模型。

## 4. 智能方法三：食材图像识别

### 4.1 当前实现功能

图像识别用于读取用户上传的食材图片，并输出结构化食材列表，供推荐与任务推进使用。

当前实际采用的是两条路线：

1. 多模态大模型识别路线
2. 本地 YOLO 检测路线

### 4.2 输入数据与输出结果

输入样例：

```text
POST /api/files/upload
POST /api/conversations/{id}/messages/stream

用户上传图片后，模型调用 recognize_image_ingredients 工具识别食材
```

输出样例：

```json
{
  "ingredients": [
    {"name": "番茄"},
    {"name": "鸡蛋"},
    {"name": "葱"}
  ]
}
```

### 4.3 训练数据集、规模与格式

对于多模态大模型路线：

- 使用已训练好的多模态大模型 API
- 不单独训练

对于 YOLO 路线：

- 基础数据集：FoodSeg103
- 补充数据：自建食材图片数据
- 当前目标类别：22 类高频常见食材
- FoodSeg103 规模：7,118 张图像，103 类
- 训练时需要筛选、映射到本项目的 22 类目标食材，再转换为检测训练格式

### 4.4 评价方法与评价指标

多模态大模型路线主要使用自建测试图片集做人工核验，指标包括：

- Precision
- Recall
- F1
- Jaccard

YOLO 路线主要使用检测模型指标：

- Precision
- Recall
- mAP@50
- mAP@50:95
- Fitness

### 4.5 智能方法在数据集上的性能评估

多模态大模型路线在当前自建测试集上的食材级结果如下：

- TP：12
- FP：2
- FN：0
- Precision：0.8571
- Recall：1.0000
- F1：0.9231
- Jaccard：0.8571

YOLO11n 路线在 FoodSeg103 的 22 类子集验证集上，第 100 轮训练后的结果如下：

- Precision：0.5294
- Recall：0.4026
- mAP@50：0.4412
- mAP@50:95：0.3502
- Fitness：0.3502

当前可以得出的结论是：

- YOLO 路线确实已经完成了训练与阶段性评测，不能视为“没有做”
- 但从当前结果看，YOLO 路线的识别效果仍弱于大模型路线，尤其在召回率和复杂场景稳定性上还有提升空间
- 因此，当前项目阶段更适合把大模型路线作为效果更好的识别方案，同时保留 YOLO 路线作为本地可控、可持续优化的训练方向
